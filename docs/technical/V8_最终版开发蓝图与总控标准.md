# V8 终版开发蓝图与阶段任务 (实施方案)

> 此方案为基于强化学习量子电路编译路由器的 V8 最终收官标准与自动执行规划。

## 一、 V8 最终验收标准
1. **极限路由质量**：基准电路集 SWAP 消耗严格 **≤ 24**（彻底超越经典 SABRE 算法的 25 SWAP 标杆）。
2. **极速推断时长**：百轮并发寻路 (`MTx100` 或深度 `MCTS`) 总系统耗时压缩至 **5 秒以内**。
3. **无悬念网络收敛**：PPO 模型在 3 万轮对局中，达到评估值 `avg_swap < 9.0` 并稳定收敛。

## 二、 三步走自动化执行策略

### 阶段一：训练解阻与全盘收敛 (Phase 1)
- 检查后台正在运行的 v8 进程日志。
- 若确认 `SWAP_PENALTY = -2.0` 带来的休克保守型塌陷，将自动重置并开启基于退火机制（Annealing Penalty）的训练，确保模型不再因为怯于行动而产生死循环。

### 阶段二：“零拷贝”引爆推断算力 (Phase 2)
- 经证实，Gym 环境中 `qiskit.QuantumCircuit` 的硬拷贝占据了 90% 以上耗时。
- 开发轻量数组型路由表 (`LightRouterState`)。在推演和前瞻展开阶段，只做原生内存数组的加减移动，摒弃庞杂面向对象拷贝。推断速度将因此暴涨 10 至 50 倍以上。

### 阶段三：跑分验证与封装部署 (Phase 3)
- V8 完全体组装完毕后自动拉起基准跑分。突破 24 SWAPs 大关后，更新所有研究报告，并将轻量引擎热插拔至 FastAPI 提供最终 API 接口。
