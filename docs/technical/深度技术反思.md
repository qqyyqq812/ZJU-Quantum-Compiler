## 一、训练动态的 6 个核心问题

### 问题 1：SWAP 数平台效应（12.5±2 不再下降）

**现象**：从 ep5000 到 ep30000+，训练 SWAP 始终在 12-15 之间震荡，没有单调下降趋势。EVAL 波动更大（9.3-20.7）。

**根因分析**：

1. **Action Mask 截断了探索空间**  
   当前 mask 逻辑（`env.py:200-209`）只保留能**立即**缩短前沿门距离的 SWAP。但最优路由可能需要"先远离再接近"（比如绕过一个阻塞节点）。SABRE 通过衰减因子（decay factor）允许这类策略，我们的 hard mask 直接禁止了。

2. **距离奖励的局部最优陷阱**  
   `reward += (dist_before - dist_after) * 0.5` 鼓励每步缩短距离。但 linear_5 拓扑是 1D 链（0-1-2-3-4），有时逻辑比特 0→物理比特 4 且 逻辑比特 4→物理比特 0，最优策略是"全部反转"（2 SWAPs），但距离奖励会鼓励一步步挪动（4+ SWAPs）。

3. **Identity Mapping 偏见**  
   训练和评测都使用 identity mapping。模型从未学过"好的初始映射长什么样"。SABRE 的双向遍历本质上就是在搜索最优初始映射。

**数据证据**：

```
训练前 10K ep: SWAP 14.5 → 14.0 (缓慢下降)
训练 10K-25K: SWAP 14.0 → 12.5 (继续下降但放缓)  
训练 25K-30K (resumed): SWAP 12.5 → 11.8 (接近平台)
```

R 从 37→42 持续上升，但 SWAP 下降速度远慢于 R 上升速度。说明模型学到了"高效利用门执行奖励"但不是"减少 SWAP"。

### 问题 2：奖励函数的信噪比问题

**当前奖励结构定量分析**（一个典型 episode）：

```
假设一个 5Q depth=5 电路, CX=10 gates:
- 需要 ~5 SWAPs 才能路由完成
- SWAP 惩罚: 5 × (-0.5) = -2.5
- 门执行奖励: 10 × 1.0 = +10.0
- 完成奖励: +10.0
- 距离缩减: ~3.0 (估计)
- 总 Reward: 10.0 - 2.5 + 10.0 + 3.0 = 20.5
```

**问题**：即使 SWAP 数从 5 变为 10（翻倍），总 reward 变化仅 `10 × (-0.5) = -5.0`，从 20.5 → 17.5（仅 -15%）。而门执行奖励 +10.0 和完成奖励 +10.0 是固定的（只要完成就拿满），这淹没了 SWAP 惩罚信号。

**修复方向**：

| 方案 | 公式 | 优势 |
|------|------|------|
| A. 加大 SWAP 惩罚 | `penalty_swap = -2.0` | 简单直接 |
| B. 降低门执行奖励 | `reward_gate = 0.3` | 让 SWAP 成为主导项 |
| C. 终端奖励 | `reward = -total_swaps` (仅完成时) | 信号最纯净 |
| **D. 相对奖励** | **`reward = sabre_swaps - ai_swaps`** | **自动校准** |

推荐 **方案 D**：先用 SABRE 路由得到基线 SWAP 数，AI 奖励 = `baseline - ai_swaps`。这样"比 SABRE 少 1 个 SWAP"就是 +1.0，信号极其清晰。

### 问题 3：Eval 波动太大（9.3 到 20.7）

**现象**：同一个模型在不同 eval 时间点的 avg_swap 从 9.3 到 20.7，差异 >2倍。

**根因**：eval 使用 **固定种子生成的 3 条电路**，样本太少。一条电路的随机性就能把 avg 拉偏 50%。

**修复**：eval 应使用 **20-50 条电路**，并报告 **中位数和标准差**，而不是平均值。

---

### 问题 4：GNN 编码器可能"看不到"关键信息

**当前节点特征**（`gnn_encoder.py` in_channels=5）：

分析 `_get_info()` 中 GNN 输入的构造（需要确认），但 5 维特征可能只包含：
- 逻辑比特 ID
- 前沿门参与状态
- 距离相关特征

**缺失信息**：
1. **逻辑-物理映射的置换信息** — GNN 不知道 "逻辑比特 3 当前在物理节点 1 上"
2. **DAG 剩余深度** — GNN 不知道还剩多少层门要路由
3. **前沿门的具体目标** — GNN 知道节点参与前沿，但不知道"节点 2 需要和节点 4 做 CX"

这些都是 SABRE 启发式函数使用的关键信息。

---

### 问题 5：PPO 的样本效率问题

**定量分析**：
- 50K episodes × ~15 steps/ep = ~750K 环境交互步
- AlphaRouter 使用 200K training steps（但配合 MCTS 搜索）
- ZX-RL 使用 100K episodes

我们的 750K 步并不算少，但 PPO 的 **rollout_steps=256** 和 **batch_size** 需要检查是否合理。如果每次 update 只用了 256 步的经验，在 5Q 电路（~15 步/episode）上意味着 ~17 episodes/update，信噪比较低。

---

### 问题 6：Identity Mapping 是隐藏的最大瓶颈

**量化分析**：Multi-Trial×10 实验已经证明：

```
Greedy (identity mapping): Total 98 SWAPs
MT×10 (random mappings):   Total 52 SWAPs  ← -47%!
```

**结论**：仅仅换一个初始映射就能减少 47% 的 SWAP！这说明：

1. Identity mapping 对于大多数电路来说**不是最优起点**
2. 模型在"错误的"初始映射上花费了大量 SWAP 来纠正
3. **初始映射优化可能比训练更多 episodes 更有效**

## 二、架构级反思

### 2.1 Edge Scorer 的局限

当前 Edge Scorer：
```python
edge_score = MLP(concat(node_i_emb, node_j_emb, global_context))
# 输入维度: 128 + 128 + 256 = 512 → 128 → 1
```

**问题**：它只看"这条边的两个端点"和"全局上下文"。它**不知道**：
- 这条 SWAP 会影响哪些前沿门
- SWAP 后映射会变成什么样
- 多个 SWAP 候选之间的相对优劣

**改进方向**：Attention over all candidate SWAPs，让模型能**比较**不同 SWAP 的优劣，而不是独立打分。

### 2.2 Observation 冗余

观测空间维度：
```
mapping_matrix:  5×5 = 25
front_matrix:    5×5 = 25  
front_distances: 20
ext_distances:   20
progress:        1
Total:           91
```

对于 5Q 拓扑 91 维不算大，但 mapping_matrix (25) 和 front_matrix (25) 是**稀疏矩阵**——大部分是 0。可以用更 compact 的表示。

### 2.3 GraphSAGE vs Transformer

| 特性 | GraphSAGE (当前) | Transformer (AlphaRouter) |
|------|-----------------|--------------------------|
| 局部感受野 | 3-hop | 全局 (self-attention) |
| 位置编码 | 隐式 (图结构) | 显式 (positional encoding) |
| 序列感知 | ❌ | ✅ (门执行顺序) |
| 计算量 | O(N×d) | O(N²×d) |
| 适用场景 | 小图 (5-20 节点) | 中大图 (20-100 节点) |

**结论**：对于 5Q 拓扑，GraphSAGE 已经足够（3-hop = 全图覆盖）。瓶颈不在 GNN 架构，而在输入特征和奖励函数。

## 三、对标分析 — 为什么 SABRE 在小规模上几乎不可能被超越

### SABRE 的 3 个结构性优势

1. **双向遍历** = 全局初始映射搜索（我们没有）
2. **精确启发式** = 基于距离矩阵的最佳 SWAP 选择（我们用 RL 近似）
3. **多次试验** = 利用随机性逃离局部最优（我们只跑一次）

### RL 的优势场景（SABRE 的弱点）

RL 真正有优势的场景是：
- **大规模拓扑**（50Q+）：SABRE 的贪心启发式 look-ahead 范围有限
- **非标准拓扑**：SABRE 对 heavy-hex 等不规则拓扑没有针对性优化
- **需要 depth-aware 路由**：SABRE 优化 SWAP 数但不一定优化电路深度

在 **5Q linear** 拓扑上，SABRE 的贪心搜索几乎是精确的（搜索空间极小，4 条 SWAP 边），RL 很难超越。
