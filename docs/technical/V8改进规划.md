# V8 量子电路 AI 路由器 — 改进规划

> 基于 V7.2 训练结果和深度文献调研，制定下一代改进方案。

---

## 一、V7.2 差距分析

### 1.1 定量差距

基于 SABRE 基线评测（linear_5 拓扑）：

| 电路类型 | SABRE SWAPs | V7.2 AI SWAPs (旧模型) | 差距倍数 |
|----------|-------------|----------------------|---------|
| random_5_d2 | 0 | 9 | ∞ |
| random_5_d3 | 2 | 15 | 7.5× |
| random_5_d4 | 3 | 11 | 3.7× |
| qaoa_5 | 2 | 3 | 1.5× |
| grover_5 | 4 | 17 | 4.3× |
| **平均** | **~3** | **~11** | **~3.7×** |

### 1.2 根因分析

通过对比 SABRE 论文和 AlphaRouter 实现，识别出 **5 个关键差距**：

#### 差距 1：初始映射质量差
- **SABRE**: 使用双向遍历（forward-backward）全局优化初始映射
- **我们**: `identity mapping`（逻辑 0→物理 0），没有任何优化
- **影响**: 初始映射差导致模型多花 2-4 个 SWAP 才能补救

#### 差距 2：奖励信号稀疏 + 噪声大
- **当前奖励**: `+1.0×门执行 - 0.5×SWAP + 0.5×距离缩减 + 10×完成`
- **问题**: SWAP 惩罚 (-0.5) 太弱，门执行奖励 (+1.0) 太强。模型倾向于"先做很多 SWAP 再执行门"而非"找最优 SWAP 序列"
- **AlphaRouter 方案**: 使用 MCTS 价值网络提供稳定的长期收益估计

#### 差距 3：缺乏多次试验 (Multi-Trial)
- **SABRE/LightSABRE**: 运行 20+ 次随机初始映射，取最优结果
- **我们**: 单次推理（greedy argmax），没有利用随机性

#### 差距 4：Action Mask 过度约束
- **当前**: 只保留能立即缩短前沿门距离的 SWAP
- **问题**: 排除了"暂时增加距离但长期更优"的 SWAP（SABRE 的衰减因子就是解决这个的）

#### 差距 5：无全局搜索能力
- **SABRE**: 贪心但带 look-ahead 启发式
- **AlphaRouter**: MCTS 树搜索
- **我们**: 纯 greedy argmax 推理

## 二、V8 改进方案（按优先级排序）

### 2.1 【高优先级】初始映射优化 — SABRE 式双向遍历

**原理**：先用 AI 模型做一次前向路由 → 取最终映射 → 反向遍历电路重新路由 → 循环 2-3 次。

**实现**：
```python
# 推理时伪代码
def ai_compile_with_bidirectional(circuit, policy, n_rounds=3):
    mapping = identity_mapping()
    for round in range(n_rounds):
        mapping, swaps = run_ai_forward(circuit, policy, initial_mapping=mapping)
        circuit_rev = reverse_circuit(circuit)
        mapping, _ = run_ai_forward(circuit_rev, policy, initial_mapping=mapping)
    return run_ai_forward(circuit, policy, initial_mapping=mapping)
```

**预期收益**: 减少 30-50% SWAP（SABRE 论文数据）  
**工作量**: ~2 小时（仅修改推理逻辑，不影响训练）

---

### 2.2 【高优先级】Multi-Trial 推理

**原理**：以不同随机映射运行 K 次 AI 路由，取 SWAP 数最少的结果。

**实现**：
```python
def ai_compile_multi_trial(circuit, policy, k=10):
    best_swaps = float('inf')
    for trial in range(k):
        mapping = random_initial_mapping()
        swaps = run_ai_forward(circuit, policy, mapping)
        if swaps < best_swaps:
            best_swaps, best_result = swaps, result
    return best_result
```

**预期收益**: 减少 20-30% SWAP（LightSABRE 论文数据）  
**工作量**: ~1 小时

---

### 2.3 【中优先级】奖励函数调优

**当前问题**：SWAP 惩罚 (-0.5) 相对于门执行奖励 (+1.0) 太弱。

**改进方案**：
| 参数 | 当前 | V8 建议 | 理由 |
|------|------|---------|------|
| `reward_gate` | 1.0 | 0.3 | 降低门执行奖励，减少"凑数"行为 |
| `penalty_swap` | -0.5 | -2.0 | 增大 SWAP 惩罚，强迫模型寻找更短路径 |
| `reward_done` | 10.0 | 20.0 | 增大完成奖励，鼓励完成路由 |
| `distance_reward_coef` | 0.5 | 1.0 | 加强距离引导 |

**额外方案 — 指数型奖励（ZX-RL 启发）**：
```python
# 基于 SABRE 基线的相对奖励
sabre_baseline = precompute_sabre_swaps(circuit)
reward = exp(-alpha * (ai_swaps / sabre_baseline))
```

---

### 2.4 【中优先级】放宽 Action Mask

**当前 mask**: 只保留能立即缩短 distance 的 SWAP。

**改进方案 — Soft Mask**：
```python
# 不是 0/1 mask，而是 score-based soft mask
mask = np.ones(n_actions)
for i, (s1, s2) in enumerate(swap_edges):
    dist_delta = compute_dist_after_swap(s1, s2) - current_dist
    if dist_delta > 2:  # 只过滤"明显变差"的
        mask[i] = 0
# 允许"距离不变"或"微微增加"的 SWAP 通过
```

**预期收益**: 模型能学到"先退一步再进两步"的策略  
**工作量**: ~1 小时

---

### 2.5 【低优先级】MCTS 混合推理

**原理**（参考 AlphaRouter）：
1. 使用 PPO 训练的策略网络作为 MCTS 的先验
2. 在推理时做浅层树搜索（depth=3-5），用价值网络评估叶节点
3. 选择搜索树中SWAP 数最少的路径

**预期收益**: 10-20% SWAP 改善  
**工作量**: ~8 小时（需要实现 MCTS 框架）

## 四、关键文献技术细节补充

### 4.1 SABRE — 局部最优逃逸

LightSABRE 引入了 **Release Valve 机制**：
- 当连续 N 个 SWAP 后前沿门距离没有缩短 → 回退最近的 SWAPs
- 使用 Dijkstra 算法找到物理最短路径强制路由
- 这是 SABRE 优于纯 RL 的关键——它不会陷入死循环

### 4.2 AlphaRouter — Transformer 编码器

- **架构**: 4 层, 6 头 Transformer encoder
- **输入**: 门序列 + positional encoding
- **对比 GraphSAGE**: Transformer 能捕获更长距离的门依赖关系
- **关键**: 训练时用 RL, 推理时用 MCTS 放大策略网络的决策能力

### 4.3 奖励设计范式

| 方法 | 奖励类型 | 优势 |
|------|---------|------|
| 即时奖励（我们） | 每步给 reward | 信号频繁，但噪声大 |
| 终端奖励 | 只在完成时给 | 信号稀疏，但准确 |
| 相对奖励 | 与 SABRE 基线比较 | 自动 calibrated |
| 指数奖励 | exp(-α×SWAP) | 对小改进更敏感 |
