# V8 冲刺升级实施方案与任务清单

> 本文档针对交接文档中提及的 V8 评测性能退化进行深度技术改版设计。

## 瓶颈诊断数据
最新 V8 Best×10 产生了 33 SWAPs，退化于 SABRE 的 25，主要缘于算法侧两点深层架构问题：
1. **启发式污染 (OOD Mapping)**：BiDir 模块带来的经典前后遍历映射并非 V8 PPO 训练中分布（Random Initial）所熟悉的环境，网络输出被误导；
2. **贪心采样陷阱**：无历史参考的多步采样极容易在一个拓扑子环中死锁（负反馈放大现象）。

## 任务拆解与自动化推进计划

- [x] 3. **算力平衡**：设定合理的 MCTS 步长与 UCB1 系数，经过性能验证后放弃厚重深拷贝，切换至 MTx100 轻量多核推断。
- [x] 4. **基准验证**：与 SABRE、V7.2 完全对齐比较，确立模型在随机初始化分布上的运行波幅（~29至35），阐明 OOD 影响范围。
