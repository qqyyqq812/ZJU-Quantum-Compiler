# 基于强化学习的量子电路编译路由优化

> **ZJU-Quantum-Compiler** — GNN + PPO 替代 SABRE 的工业级路由器  
> 最后更新: 2026-03-26

---

## 1. 研究背景与问题定义

量子计算中，逻辑电路必须映射到物理芯片上执行，但超导量子比特的连接拓扑是受限的（Coupling Map）。**路由问题**即在拓扑约束下用最少 SWAP 操作完成电路映射——这是一个 NP-Hard 问题。

当前工业标准算法 **SABRE**（Li et al., ASPLOS 2019）基于启发式前/后向遍历，虽然高效，但存在四大局限：
1. 贪心决策缺乏全局规划
2. 无法学习拓扑特异性特征
3. 固定评分函数缺乏自适应性
4. 不同芯片拓扑需单独调参

本项目探索用 **图神经网络 (GNN) + 近端策略优化 (PPO)** 将路由决策建模为马尔可夫决策过程 (MDP)，训练端到端的 AI 路由器。

## 2. 技术方案

### 2.1 系统架构

```
输入: 量子电路 (QASM/Qiskit) + 目标拓扑 (CouplingMap)
         │
    ┌────▼────┐
    │ CircuitDAG │  ← 前沿层提取 + 依赖关系分析
    └────┬────┘
         │
    ┌────▼────────────────┐
    │ RL Environment (V3) │  ← Gymnasium 接口, Action Masking
    │  State: 291 维拓扑无关编码    │
    │  Action: SWAP 边 + PASS      │
    └────┬────────────────┘
         │
    ┌────▼──────────────────┐
    │ PolicyNetwork (V7)     │
    │  GraphSAGE 3 层编码器  │
    │  Edge-Scoring Attention │
    │  独立 Critic 头         │
    └────┬──────────────────┘
         │
    ┌────▼───────────┐
    │ PPO Trainer     │  ← Clip=0.2, GAE λ=0.95
    │  + 课程学习调度  │  ← 5 阶段 3Q→20Q
    │  + LR 余弦退火   │
    └────────────────┘
```

### 2.2 核心技术点

#### (a) V9 拓扑无关观测编码 (obs_dim=291)

传统方法用 N×N 映射矩阵编码观测 (5Q=71 维, 20Q=841 维)，导致网络无法跨拓扑迁移。V9 改为**边驱动**固定维度编码：

| 特征组 | 维度 | 描述 |
|--------|------|------|
| SWAP 边特征 | 60×4=240 | 每条边 4 维: best/worst/mean delta, n_improved |
| 前沿门距离 | 20 | 当前前沿层双比特门物理距离 |
| Look-ahead 距离 | 20 | 后续层门距离 |
| 聚合统计量 | 10 | front_gates, sum/mean/max/min dist, edges, N, swaps, steps, remaining |
| 进度 | 1 | 剩余门比例 |
| **总计** | **291** | **与 N 无关，5Q/20Q 共用** |

#### (b) GNN Actor-Critic (V7)

- **特征提取**: 3 层 GraphSAGE 编码物理图节点特征 (128 维)
- **融合**: 扁平 obs (128) + 图全局池化 (256) → 384 → 256 维全局上下文
- **Actor**: Edge-Scoring Attention 头 — 对每条 SWAP 边独立打分 (src 128 + dst 128 + ctx 256 = 512 → 1)
- **Critic**: 独立价值网络 (256 → 128 → 1)，不受 Actor 高频梯度干扰
- **PASS**: 独立 PASS Scorer (ctx 256 → 64 → 1)

#### (c) 5 阶段课程学习

| 阶段 | 名称 | 比特数 | 电路深度 | 升级阈值 (SWAP) |
|------|------|--------|----------|-----------------|
| 0 | warm-up | 3 | 1-2 | 2.0 |
| 1 | elementary | 5 | 2-4 | 8.0 |
| 2 | standard | 5 | 4-6 | 12.0 |
| 3 | challenge | 10 | 6-10 | 25.0 |
| 4 | master | 20 | 10-20 | 50.0 |

自适应宽限机制：卡住超过 5000 episodes 后逐步放宽阈值至 2 倍。

#### (d) V8 推断引擎

经实验验证，抛弃树搜索 (Beam Search / MCTS)，确立 **MTx100 (多起点贪心采样)** 为终极推断核心：
- 树搜索中 PyTorch forward 调用 30000+ 次，实用性差
- MTx100: 100 次独立随机映射 + 单次 forward pass 贪心推理，取最优

#### (e) 零拷贝环境 (LightweightEnv)

解决 Qiskit `QuantumCircuit` 深拷贝 ~10ms/次的性能瓶颈，纯 Numpy 数组状态机实现 O(1) clone。

## 3. 训练结果

### 3.1 V7.2 (5Q linear)

- **训练**: 50000 episodes, 课程学习 (3Q→5Q)
- **结果**: avg_swap = 9.3, 已收敛
- **模型**: `models/v7_final_v2/v7_linear_5_best.pt`

### 3.2 V8 (5Q linear, 随机映射)

- **训练**: Early Stop @ep17000
- **改进**: 随机初始映射增强泛化性
- **奖励函数**: gate=1.0, swap=-1.0, done=20.0 (惩罚从 -2.0 回退至 -1.0)
- **结果**: avg_swap = 13.0 (随机映射下)
- **模型**: `models/v8_rewarded/v7_linear_5_best.pt`

### 3.3 V9 (20Q IBM Tokyo) — 进行中

- **拓扑**: IBM Tokyo 20Q (43 条双向边, 直径 4, 平均度 4.3)
- **训练状态**: 中断于 ep5000, Stage 4 master (20Q)
- **当前水平**: SWAP ~91-106, EVAL done=0% (20Q 电路在 max_steps=500 下可能不够)
- **待恢复**: checkpoint 位于 `models/v9_tokyo20/checkpoint_ep5000.pt`

## 4. 关键技术决策记录

| 决策 | 理由 |
|------|------|
| 抛弃 Beam Search / MCTS，选择 MTx100 | 树搜索中 PyTorch forward 调用次数爆炸 (30000+次)，不如 100 次独立贪心采样 |
| obs 从 N² 映射矩阵改为边驱动 291 维 | 使同一套网络权重可跨 5Q/9Q/14Q/20Q 拓扑迁移，课程学习真正可行 |
| V8 SWAP 惩罚从 -2.0 退回 -1.0 | -2.0 导致策略完全丧失探索能力，模型选择持续 PASS 死锁 |
| 选择 IBM Tokyo 而非 Grid 5x4 | Tokyo 是真实芯片拓扑，有缺失边和非均匀度，论文价值更高 |
