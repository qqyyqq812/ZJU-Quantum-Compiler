# 📚 学术调研报告：量子电路路由中的强化学习“震荡（Ping-Pong）”规避策略

> **调研目标**：探究在放宽动作掩码（Soft Mask）的前提下，前沿强化学习路由算法如何避免模型陷入无意义的来回互换（Infinite Loops / Oscillations）死锁，并提出针对本项目 V9 架构（20Q 真实拓扑）的最优解决路径。

## 一、现象剖析：为什么会“打乒乓”？

在我们的第一次冒烟测试中，放开硬掩码后 SWAP 次数从 26 爆退到 443。此现象在学界极其经典。
在早期的纯 PPO 路由论文（如基于 Qiskit SABRE 的 RL 探索）中，研究者指出：由于量子态的马尔可夫决策过程（MDP）中，**状态空间的极其庞大与奖励的极度稀疏**，如果允许模型执行“短期内距离退步”的动作，模型极易陷入局部最优的循环中。
通俗来说：模型发现“前进”的奖励被锁死（因为遇到了复杂的网络瓶颈），而“倒退”虽然没有奖励，但紧接着走回原位的动作又能获得“距离缩减”的奖励，于是模型学会了通过不断“原地鬼畜”来刷取虚假的内部微小波动。

## 二、前沿（SOTA）解决方案综述

通过对最新学术论文的文献检索，工业界和学术界解决该困局主要形成两大技术流派：

### 流派 A：环境改造派 —— 禁忌记忆法 (Tabu-RL)
此流派的思想来源于经典的“禁忌搜索（Tabu Search）”。
**实现原理**：在 RL 环境 `env.py` 中引入一个“短期记忆窗口（History Buffer）”。例如记忆最近 3 步执行过的 SWAP 边。如果当前动作企图执行一个在记忆表里的逆向动作（即刚换了 A-B，又要换 B-A），环境直接给予毁灭性的巨大惩罚（Penalty = -10.0），并在后续几步内强行 Mask 掉这个动作。
- **优点**：极快！不怎么增加推理耗时，非常契合我们目前的 `MTx100` 高并发极速引擎。
- **缺点**：过于硬核的规则干预，可能会意外杀掉某种真的需要“退两步进三步”的终极神仙走法（假阳性过高）。

### 流派 B：树搜索演化派 —— 前瞻寻路 (MCTS / Beam Search)
这也是目前 DeepMind 在 AlphaTensor 和各类巨型图路由任务上最爱用的 SOTA 体系。
**实现原理**：抛弃简单的“贪心摸着石头过河（Greedy Rollout）”或“蒙眼抽卡（Multi-Trial）”，将 RL 网络降级为“启发向导（Heuristic Prior）”。推断时，程序在内存中维护一棵“决策树”，每走一步都要往后看 3 步（Beam Width）。由于同一棵树上的节点会严格检查是否成环（Cycle Detection），且循环节点的综合概率会被剪枝，因此从物理数学机制上彻底杜绝了乱走。
- **优点**：能够发掘极其深远、反直觉的神仙走法，上限极高。对 20Q 这种大拓扑具有降维打击力。
- **缺点**：计算开销极其恐怖。每一步都要 deepcopy 环境（由于 Qiskit DAG 图对象极难深拷贝，可能导致推断比 SABRE 慢上千倍），极易 OOM（内存爆满）。

## 三、我们的科研向对比实验设计 (Ablation Study)

作为一项追求严谨的科研工作，我们不能仅凭理论拍脑袋决定。我计划设计如下的 **A/B 架构大比武（对比实验）**：

**【实验变量配置】**
- **Baseline (基线)**: SABRE 算法。
- **Candidate 1 (候选方案一)**: 引入 `Tabu_Size=3` 的记忆环境 + `Soft_Mask=True` + `MTx100` 极速并行引擎。
- **Candidate 2 (候选方案二)**: 启用我在 `inference_v8.py` 中预留但尚未调试成熟的 `Beam Search` 引擎 + `Beam_Width=3` + `Soft_Mask=True`。

**【评测指标 (Metrics)】**
我们将在 `docs/qa_supervision/benchmark_tracking.md` 中记录双维度数据：
1. **SWAP 数绝对值**（衡量路由质量上限，越低越好）
2. **单路编译耗时 (Time)**（衡量方案在工业界的落地可行性，越快越难得）

我们将使用包含 QFT、QAOA 等的 5Q 测试集做先锋对决，胜出的一方，将被我们定为 V9 架构的最终范式，并以此为基础进军 20Q 极限训练！
