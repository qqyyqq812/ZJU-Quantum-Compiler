## 五、怎么跑

```bash
# 进入环境
cd ~/projects/量子电路 && source .venv/bin/activate
export OMP_NUM_THREADS=1 MKL_NUM_THREADS=1

# 评测（V7.2 最终模型）
python -u -m src.compiler.inference_v8 models/v7_final_v2/v7_linear_5_best.pt

# 评测（V8 训练完成后）
python -u -m src.compiler.inference_v8 models/v8_rewarded/v7_linear_5_best.pt

# 训练曲线
python -m src.benchmarks.plot_training models/v7_final_v2/history_v7_linear_5.json results/v7_final

# 启动后端
uvicorn src.server.app:app --port 8000
```

> ⚠️ **必须** `OMP_NUM_THREADS=1 MKL_NUM_THREADS=1`，否则 PyTorch+Qiskit 死锁。

## 九、用户开发偏好与核心准则

新对话框接手后，务必遵守以下用户偏好与开发准则：

1. **简洁至上 (KISS 原则)**：代码崇尚简洁与可维护性，避免过度工程化（例如：避免创建不必要的中间层、过度复杂的抽象）。
2. **第一性原理与深度分析**：遇到 Bug 或需要引入新框架时，优先剖析根本原因。能够使用 `find-skills`、`rg`、`python -m cProfile` 等工具找出瓶颈，而不是盲目改代码。
3. **事实为本**：汇报任何性能提升（例如 SWAPs 数量改善、速度提升）必须要有定量的数据支撑。如有谬误，需要坦率纠正。
4. **自动化与端到端开发**：
   - 具备高度自治能力，善于利用已有的 Skills（如 `npx skills`、代码评审插件等）自动化完成任务。
   - 制定好完整方案后，倾向于顺畅地端到端自驱动完成代码修改与评测，不需要在琐碎步骤处频繁停顿。
   - 具体执行时，自行分解为明确的具体任务（Tasks/Checklists），逐个击破并自我验证。
5. **沟通与文档规范**：
   - 必须全中文沟通与思考（`Implementation Plan, Task List and Thought in Chinese`）。
   - 当涉及具体文件内容的修改时，需要给出绝对链接 `[文件名](file:///absolute/path/to/file)`，便于用户直接点击查看。
   - 回答用户的问题需逐个分点详细解答（不能遗漏）。
   - 所有创建的文档必须使用**中文命名** `.md` 文件（如 `系统架构.md`）。
