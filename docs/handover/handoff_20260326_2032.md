### 2. max_steps 瓶颈修复
- **问题**: 20Q 电路在 `max_steps=500` 下大量 truncated, EVAL done=0%→33%
- **修复**: `env.py` 默认值 500→2000
- **同时修复**: `train.py` evaluate_model 中硬编码的 500 步上限改为 `env.max_steps`

### 3. Checkpoint 完整状态保存
- **问题**: 旧 checkpoint 仅保存 `policy.state_dict()`，恢复后从 Stage 0 重新爬升
- **修复**: 新格式保存 `{model_state, optimizer_state, episode, curriculum_stage, best_eval_swaps, patience_counter}`
- **兼容**: 恢复时自动识别新旧格式；旧格式自动跳到最终课程阶段

### 4. load_policy 跨版本兼容
- **问题**: V7/V8 模型 obs_dim=91, V9 env obs_dim=291, 直接加载报 shape mismatch
- **修复**: 从 checkpoint 的 `obs_encoder.0.weight.shape[1]` 自动推断 obs_dim
- **注意**: 返回值改为 `(policy, obs_dim)` 元组

### 5. GitHub 全量推送
