## 一、这个项目是什么

**一句话**：用 AI（强化学习）优化量子电路在真实量子芯片上的编译路由，让插入的额外 SWAP 门尽可能少，目标超越 IBM 的 SABRE 算法。

**类比**：传统编译器（如 GCC）把 C 代码编译到 x86 指令集时需要做寄存器分配。量子编译器也一样——把逻辑量子电路"编译"到物理芯片上时，需要根据芯片的连接限制插入 SWAP 门搬运数据。SABRE 是目前业界的标准算法（相当于 GCC -O3），我们训练的 AI 要比它更优。

**目标芯片**：IBM Heavy-Hex 拓扑（IBM Eagle 127-qubit 芯片的架构）。训练从小规模 5-qubit 开始，通过课程学习自动升级。

## 三、怎么跑

### 日常开发（进入环境）
```bash
cd ~/projects/量子电路
source .venv/bin/activate
```

### 训练模型
```bash
# 快速测试（30 秒）
python -m src.compiler.train --topology linear_5 --episodes 500 --curriculum --save-dir models/test

# 正式训练（2 小时）
export OMP_NUM_THREADS=1 MKL_NUM_THREADS=1   # 防止 PyTorch/Qiskit C++ 库冲突
python -u -m src.compiler.train --topology linear_5 --episodes 50000 --curriculum --save-dir models/v7_final 2>&1 | tee training.log
```

### 评测模型
```bash
export OMP_NUM_THREADS=1 MKL_NUM_THREADS=1
python -c "
import faulthandler; faulthandler.enable()
import src.benchmarks.run_v7_eval as ext
ext.run_benchmark_suite('models/v7_final/v7_linear_5_best.pt', 'linear_5', 5, 'results/final')
"
# 查看报告
cat results/final/evaluation_v7_linear_5.md
```

### 启动 Web 后端
```bash
python -m src.server.app
# 访问 http://localhost:8765/static/
```

> **⚠️ 重要**：运行评测和训练时，必须设置 `OMP_NUM_THREADS=1 MKL_NUM_THREADS=1`，否则 PyTorch 和 Qiskit 的底层 C++ 库会冲突导致进程静默崩溃。

## 六、用户偏好与约定

1. **纯结果导向**：用户不需要理解算法原理，只关心最终效果和报告质量
2. **文献综述是评分重点**：老师大作业要求"利用 AI 大模型理解前沿文献中的算法框架"
3. **中文输出**：所有文档、思考过程、task list 均使用中文
4. **文档命名**：`.md` 文件使用中文命名
5. **先方案后执行**：任何改动先提出方案，获得用户同意后再执行
6. **简洁至上**：KISS 原则，避免过度工程化
7. **硬件限制**：本机 i5-13500H + 16GB + 无 GPU，Colab Pro 不可用（家庭组成员无权限）
8. **训练方式**：本机 CPU 多核运行，用 `tee` 而不是 `nohup`（方便实时看进度）
