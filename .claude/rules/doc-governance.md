# 文档治理规则 (量子电路)

## 禁止生成的文档类型

严禁生成以下文档：

1. **AI 工作交接文档**
   - 禁止：`handoff_*.md`, `交接_*.md`, `03_AI协同日志.md`

2. **规划与执行文档**
   - 禁止：`EXECUTION_PLAN_*.md`, `FINAL_SPRINT_*.md`
   - 禁止：`AutoDL_V*.md`（历史验收报告）

3. **版本迭代文档**
   - 禁止：`V[n]改进规划.md`, `V[n]改进实施方案.md`, `V[n]_最终版*.md`
   - V8-V12 历史文档应删除，新决策写入 `decisions.md`

4. **临时调研报告**
   - 禁止：`*调研.md`, `*前沿.md`, `SOTA_*.md`
   - 例：`量子路由强化学习前沿调研.md` 应删除，关键结论提炼到 `decisions.md`

5. **工作日志与反思**
   - 禁止：`深度技术反思.md`, `Workflow_*.md`（运维日志）

## 永久文档标准

### 核心保留

| 文档 | 用途 | 更新频率 |
|-----|------|---------|
| `decisions.md` | 版本决策 + 踩坑记录 | 每当算法改变时 |
| `01_物理基础.md` | 量子电路基础 | 偶尔 |
| `03_SABRE精读.md` | 算法细节 | 偶尔 |
| `05_文献综述.md` | 论文参考 | 定期更新 |
| `colab_workflow_and_pitfalls.md` | **Colab 训练踩坑** | 重要保留 |
| `项目报告.md` | 论文初稿 | 定期更新 |
| `cc-updates.md` | AI 自动维护 | 自动 |

### 笔记本约定

- 提交笔记本前**必须清除所有单元输出**
- 只保留关键的配置和验证笔记本
- 训练日志应存到 `models/logs/` 而非笔记本内

---

## 执行规则

1. 临时文档 → `/tmp/` 或 Colab session，不提交项目
2. 发现禁止文档 → 提炼关键内容到 `decisions.md` 后删除
3. 每个工作周期结束时 `find docs -name "*.md" | wc -l` 确保 < 15 个
